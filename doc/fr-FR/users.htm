Introduction

Hyla possède un système de gestion des utilisateurs, qui couplé au système de droits permet de définir des restrictions par dossier.

Il vous sera également possible de définir des groupes d'utilisateurs dans lesquelles, vous pourrez rassemblez des utilisateurs, simplifiant alors grandement l'attribution de droits.

Convention de nommage

Un nom d'utilisateur ou un nom de groupe peut contenir n'importe quelle lettre de l'alphabet, des chiffres, le trait d'union ( - ) et le tiret bas ( _ ) dans la limite de 32 caractères et le nom doit impérativement commencer par une lettre.

Enfin, Hyla convertira le nom entièrement en minuscule évitant ainsi toute ambiguïté.

Les utilisateurs spéciaux ou méta-utilisateurs

Par défaut, il existe trois utilisateurs spéciaux, aussi appelé méta-utilisateurs, ces derniers permettent de sélectionner des utilisateurs, parfois, selon leur statuts de connection.

  • Any qui définit n'importe quel utilisateur sans distinction.
  • Authenticated qui définit tous les utilisateurs ayant un statut connecté, c'est à dire authentifié.
  • Anonymous qui définit tous les utilisateurs anonymes.

Ces utilisateurs ne peuvent pas être supprimés.

Les différents types d'utilisateurs

Un utilisateur peut être défini à sa création en trois types :

  • Standard : Aucun droit particulier
  • Superviseur : Un superviseur possède tous les droits sauf celui d'accéder à l'administration et d'éditer les droits, il peut également accéder à tous les dossiers, qu'il soit protégés par des droits ou non.
  • Administrateur : L'administrateur, possède, bien sûr, tous les droits.

Les groupes

Un groupe peut contenir n'importe quel utilisateur, sauf, les méta-utilisateurs.

La convention de nommage pour les groupes est la même que pour les utilisateurs.

Un groupe est réprésenté entre crochet (exemple: [Test]).
