Configuration

Hormis les informations de configuration contenues dans le fichier /conf/config.inc.php, Hyla stocke les informations restantes dans le fichier /conf/hyla.ini.
Ce dernier est, comme son l'extension l'indique, un fichier au format de stockage .ini, c'est à dire, qu'il est très facilement éditable grâce à un éditeur de texte, ainsi, il vous sera possible de modifier la configuration de Hyla par le biais de l'interface graphique de l'administration ou directement ou ouvrant ce fichier avec votre éditeur de texte favoris.

Remarque : Si par mégarde, vous faisiez une erreur en éditant un des fichiers de configuration (/conf/config.inc.php ou /conf/hyla.ini), sachez qu'il existe des versions de secours pour ces deux fichiers, il s'agit respectivement de /conf/config.inc.default.php et de /conf/hyla.default.ini), il vous suffit de les copier et de les renommer.

L'interface de configuration de l'administration est divisé en quatre parties, nous allons les aborder une par une ci-dessous en détaillant les options possibles.

Affichage

  • Titre navigateur : Il s'agit du texte mis entre les balises "title" qui apparaîtra sur la barre de titre du navigateur.
  • Le modèle par défaut : Il s'agit du dossier dans (/tpl) contenant le modèle définissant l'apparence.
    Choisissez le style par défaut contenu dans le modèle voulu, ces derniers apparaissent en gras.
  • Faut-il toujours voir les actions ? Permet d'afficher les liens vers les actions de la barre d'outils même lorsque l'on n'est pas connecté.
  • Faut-il afficher l'arborescence des dossiers ? Ceci permet d'afficher tous les dossiers sous forme d'arborescence.
  • Faut-il afficher les fichiers cachés ? En autorisant l'affichage des fichiers cachés, vous afficherez les fichiers commençant par un "." .

Ajout de fichiers et dossiers

  • Droits à attribuer aux fichiers uploadés : Lors de l'ajout d'un fichier dans Hyla, il s'agit des droits qui lui seront attribué (prenez garde en modifiant cette option !).
  • Droits à attribuer aux dossiers créés : Lorsqu'un dossier est créé, il s'agit des droits qui lui seront attribué (prenez garde en modifiant cette option !).
  • Action à réaliser lors de l'envoi d'un fichier anonyme : Pour chaque fichier anonyme envoyé, il est possible de recevoir un courriel vous avertissant de l'ajout d'un nouveau fichier à valider.

Listage de dossiers

  • Tri par défaut : Ce tri définit l'ordre d'affichage des fichiers et des dossiers.
  • Faut-il mettre les dossiers en premier ? Affiche les dossiers en premier.
  • Grouper par critère de tri : Si le tri courant est alphabétique, les fichiers commençant par la lettre A seront groupés ensemble, suivis des fichiers commençant par la lettre B, et ainsi de suite...et cela de manière dynamique, par rapport au critère de tri.
  • Nombre d'objets par page : Pour afficher tous les fichiers et dossiers, indiquez 0 mais cela est FORTEMENT non recommandé pour des raisons de charge serveur dans le cas de génération de galeries, vous êtes prévenu !!!
  • Plugin des dossiers par défaut : Spécifiez ici le plugin par défaut à utiliser, ainsi, si vous désirez générer uniquement des galeries photos avec le plugin zenphoto, choisissez "zenphoto".

Divers

  • Courriel du webmestre : Les notifications (ajout de fichiers anonymes...) seront envoyées à cette adresse.
  • Langue : Assurez-vous qu'un dossier contenant les fichiers adéquates se trouvent bien dans le dossier l10n/ .
  • Faut-il activer le compteur de téléchargement ? Cette option vous permet de compter les téléchargements et non les visualisations.
  • Le temps de redirection : Paramétrez ici le temps de redirection entre les pages exprimé en seconde, la valeur minimale est de 1 seconde.
  • Faut-il autoriser le téléchargement de dossier ? Cette option vous permet de désactiver le téléchargement de dossier qui consiste à envoyer une archive contenant les fichiers de ce dernier.
  • Le système de fichier est-il encodé en Utf8 ? Si lors d'opérations (renommage, création de dossier...) sur le système de fichiers par le biais de Hyla, apparait des caractères bizarres dans le nom des fichiers ou dossiers, modifiez cette clef de manière à ce qu'elle soit à Oui pour un système de fichiers en Utf8 ou à non pour les autres types (ISO...), si vous êtes utilisateur de Windows, cette clef devrait être à Non.
  • Autoriser les utilisateurs à créer des comptes ? Si cette option est positionnée sur "oui", un utilisateur pourra créer de lui même son propre compte utilisateur.

Utilisateurs

Pour plus d'informations concernant les utilisateurs dans Hyla, référez-vous à la documentation sur les utilisateurs et les groupes

Hyla vous permet de gérer plusieurs utilisateurs, ainsi, il est possible de restreindre l'accès à certain utilisateurs, ou encore autoriser des utilisateurs à ajouter, éditer, supprimer...

Liste des utilisateurs

Ce tableau vous permet de voir tous les utilisateurs du système et leur type.

Trois types d'utilisateurs peuvent être créé :

  • Utilisateur standard qui ne dispose d'aucun droit particulier.
  • Superviseur qui dispose de tous les droits sauf celui d'accéder à l'administration.
  • Administrateur possède tous les droits, prenez garde à qui vous attribuez ce type.

Vous pourrez remarquer la présence de 3 utilisateurs ayant un type "Spécial" appelés aussi "méta-utilisateurs", ils appartiennent au système, pour cette raison, il est impossible de les éditer, des les supprimer ou, bien sûr, d'en ajouter, voici leur rôle :

  • Any correspond à n'importe quel utilisateur, connecté ou non.
  • Anonymous correspond à toute personne non connectée (qui n'est pas authentifiée)
  • Authenticated correspond à toute personne connectée (qui est donc authentifiée)

Ajout / Modification

Pour ajouter un utilisateur dans la système, il vous suffit de cliquer sur le lien adéquat et de remplir les champs suivants :

Nom
Le nom d'un utilisateur ne peut être constitué que de lettres de l'alphabet, du trait d'union (-) et du tiret bas (_), attention tout de même, le nom doit commencer par une lettre et est limité à 32 caractères.
Mot de passe
Le mot de passe de l'utilisateur ne peut pas être vide, il peut contenir n'importe quel caractère.

Chaque utilisateur créé est par défaut un utilisateur "standard", pour changer sont type, une fois créé, éditez son type.

Suppression

Dans la colonne "Actions", vous trouverez en face de chaque utilisateur, sauf utilisateurs spéciaux et vous même, un lien permettant de supprimer l'utilisateur.

Groupes

Pour plus d'informations concernant les groupes dans Hyla, référez-vous à la documentation sur les utilisateurs et les groupes

Afin de rendre la gestion des utilisateurs plus simple, Hyla vous permet de créer des groupes d'utilisateurs permettant ainsi une manipulation des droits beaucoup plus simple.

Liste des groupes

Ce tableau vous permet de voir tous les groupes du système, leur id (identifiant unique) et les utilisateurs leur appartenant.

Ajout

Pour ajouter un groupe, il vous suffit de cliquer sur le lien adéquat et de remplir le champ suivant :

Nom
La règle de nommage des groupes est la même que pour les utilisateurs, il ne peut être constitué que de lettres de l'alphabet, du trait d'union ( - ) et du tiret bas ( _ ), attention tout de même, le nom doit commencer par une lettre et est limité à 32 caractères.

Modification

La modification d'un groupe se borne à l'ajout ou la suppression d'utilisateur.

Lors de l'ajout, une liste vous propose de choisir les utilisateurs à ajouter.

Lors de la suppression d'un utilisateur, un tableau vous permet de sélectionne par le biais de case à cocher les utilisateurs à supprimer.

Suppression

Pour supprimer un groupe, cliquez simplement sur le lien prévu à cet effet.
Le groupe sera alors supprimé ainsi que toute ces références dans les droits.

Droits

Pour plus d'informations concernant les droits dans Hyla, référez-vous à la documentation consacrée aux droits.

Liste des droits

Ce tableau vous permet de voir tous les droits du système en une seule vue.

Ce dernier est composé de quatre colonnes :

  • Dossiers Affichage du nom du dossier possédant le droit courant.
  • Utilisateurs / Groupes Cette colonne affiche les utilisateurs ou groupes ayant des droits dans le dossier courant.
  • Droits La liste de droits disponible pour les utilisateurs ou groupes du dossier courant.
  • Actions Dans cette colonne, vous disposez de l'action "Ajouter" permettant d'ajouter des droits au dossier courant et le lien "Éditer" permettant d'ajouter ou d'enlever des droits dans le dossier courant.
  • Sélection Cette colonne vous permet de sélectionner, par le biais de case à cocher, des droits afin de les supprimer grâce au bouton adéquate.

Ajout d'un droit

Pour ajouter un droit, il vous suffit de cliquer sur le lien adéquat, la création d'un droit se fait alors en deux étapes :

  1. Choix d'un dossier Dans cette première étape, un menu vous permet de choisir le dossier sur lequel vous voulez ajouter un droit.
  2. Choix d'utilisateur et attribution des droits Dans la deuxième étape, vous pouvez alors choisi un oui plusieurs utilisateurs ou groupes et leur assigner les droits désirés.

Remarque : Dans la deuxième étape, vous ne verrez afficher que les utilisateurs ou groupes ne faisant pas déjà parti des droits du dossier édité.

Édition d'un droit

En cliquant sur le lien "Éditer" dans le tableau, vous pourrez ainsi modifier les droits de(s) utilisateur(s) / groupe(s) du dossier courant.

Remarque : N'oubliez pas que les droits que vous attribuez à un utilisateur ou un groupe se propagent dans les dossiers fils.
Pour plus d'informations à ce sujet, veuillez consulter la documentation consacré au droits.

Suppression d'un droit

En cliquant sur le lien "Supprimer" dans le tableau, vous supprimerez les droits de(s) utilisateur(s) / groupe(s) du dossier courant.

Commentaires

Il vous est possible de supprimer de voir tous les commentaires et de les supprimer un par un en cliquant sur la corbeille ou par bloc en les sélectionnant grâce aux cases à cocher puis en cliquant sur le bouton "Supprimer la sélection".

Fichiers anonymes

Le tableau affiche tous les fichiers anonymes qui ont été envoyés et qui sont en attente d'une validation.

Si vous souhaitez accepter le fichier, il vous suffit de cliquer sur le lien adéquate, le fichier sera alors déplacé vers sa destination.

Maintenance

Cette partie rassemble toutes les actions utiles pour le bon fonctionnement de Hyla.

Vider le cache

Cette action vous permet de supprimer tous les fichiers et dossiers cache (/sys/cache), ce qui peut être utile si des erreurs de lecture de contenu d'archives se produisent.

Lancer une synchronisation

Si vous supprimez des fichiers ou des dossiers sans passer par l'interface de Hyla (ftp...), il peut être utile d'effectuer une synchronisation de la base de données avec le système de fichiers.
